# Codex Instructions

## Project

Sattva Path Collective is currently a fresh workspace. Keep changes small,
readable, and easy to run from Windows PowerShell.

## Working Style

- Inspect the existing files before editing.
- Prefer focused changes over broad rewrites.
- Use `rg` for searches when it is available.
- Do not commit changes unless the user explicitly asks.

## Verification

- Run the most relevant local check after changes.
- If a check cannot run because a dependency, login, or network access is
  missing, say that clearly in the final response.
